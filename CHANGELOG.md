# 更新日志

所有版本的变更记录。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/)。

## [0.5.6] - 2026-07-07

### 新增
- AI 设置页新增供应商预设：内置 OpenAI、通义千问、DeepSeek、月之暗面 Kimi、智谱 GLM、文心一言、OpenCode Go 等 8 家供应商，切换时自动填入官方 base URL，并附带文档链接
- 新增"获取模型"按钮：基于 base URL + API Key 自动拉取模型列表填充到模型输入框
- 新增"测试连接"按钮：在保存配置前对当前选中的模型发起一次最小调用验证连通性

### 优化
- AI 设置页供应商字段下拉选择，模型字段改为可补全的自由输入
- 后端新增 `ai_chat` / `ai_fetch_models` / `ai_test_connection` 三个 Tauri 命令（基于 reqwest），前端 AI 设计、AI 写 SQL 等调用点统一迁移至后端，绕过 WebView 跨域与 CORS 限制

---

## [0.5.5] - 2026-06-25

### 优化
- 调整项目详情页表列表的展示格式：表名（英文）作为主要信息（粗体）置于第一行，中文名作为次要信息（灰色小字）置于第二行，两者均单行展示，超出使用省略号，悬停显示完整内容
- 表结构区域顶部标题调整为「表名(中文名)」格式，与表列表的主次信息顺序保持一致；中文名为空时自动隐藏括号

---

## [0.5.3] - 2026-06-17

### 修复
- 修复生成同步脚本时未包含"说明"字段（comment）变化的问题：当本地列的说明为空而远程有说明时，原本不会生成任何清除注释的 SQL，现在会为 PostgreSQL / Oracle 生成 `COMMENT ON COLUMN ... IS ''` 来同步清空远程注释

---

## [0.5.2] - 2026-06-17

### 新增
- PostgreSQL 方言新增 datetime → timestamp without time zone 映射

### 修复
- 修复 PostgreSQL 远程同步时 int 与 integer 被误判为类型差异的问题；比较逻辑改用方言类型别名规范化，同时 SQL 导出时 int 正确映射为 INTEGER
- 修复数据库同步-编程对象 Tab 中多处翻译 key 不匹配/缺失导致显示原始 key 的问题
- 修复生成同步脚本时未包含索引变更的问题；新建表时同步创建索引，修改表时按差异状态生成 CREATE INDEX / DROP INDEX / 重建索引
- 修复 PostgreSQL 同步脚本中 nullable 列多余输出 DEFAULT NULL 的问题；PostgreSQL 方言不再输出 DEFAULT NULL，ALTER COLUMN 场景改为 DROP DEFAULT
- 修复 SQL 导出/同步脚本中函数默认值（如 current_timestamp、now()）被错误加引号的问题
