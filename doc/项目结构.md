# DB Designer 项目结构

> Tauri v2 桌面应用：React (TypeScript) + Rust，用于可视化设计数据库表结构并生成 SQL。

```
db-designer/
├── doc/                          # 项目文档
├── src/                          # 前端源码 (React + TypeScript)
├── src-tauri/                    # 后端源码 (Rust + Tauri)
├── index.html                    # Vite 入口 HTML
├── package.json                  # 前端依赖与脚本
├── tsconfig.json                 # TypeScript 配置
├── tsconfig.node.json            # Node 环境 TS 配置（vite.config 等）
└── vite.config.ts                # Vite 构建配置
```

## 后端 `src-tauri/`

```
src-tauri/
├── Cargo.toml                    # Rust 依赖配置（tauri, rusqlite, mysql, postgres, serde 等）
├── tauri.conf.json               # Tauri 应用配置：窗口、安全策略、自动更新、打包设置
├── build.rs                      # Tauri 构建脚本
├── capabilities/
│   └── default.json              # Tauri 权限能力配置
└── src/
    ├── main.rs                   # 程序入口，调用 lib::run()
    ├── lib.rs                    # 应用核心：注册所有 Tauri 命令、初始化插件
    ├── db.rs                     # SQLite 数据库管理：初始化连接、建表迁移、数据目录管理
    ├── models.rs                 # 数据模型定义：Project, TableDef, ColumnDef, IndexDef, Version, Snapshot, RemoteTable, DatabaseConnection 等
    ├── dialect.rs                # 数据库方言抽象层：DatabaseDialect trait（SQL 生成）+ DatabaseConnector trait（远程连接）+ MySQL/PostgreSQL 实现 + 类型映射 + 工厂函数
    ├── project.rs                # 项目 CRUD：get_projects, create_project, delete_project
    ├── table.rs                  # 表结构 CRUD：表的增删查、列定义保存、索引管理、初始数据管理
    ├── setting.rs                # 设置键值存储：get_setting, save_setting, get_all_settings
    ├── db_connection.rs          # 数据库连接配置 CRUD：连接信息的增删改查
    ├── version.rs                # 版本管理与 SQL 生成：创建版本快照、导出版本 SQL、生成版本间升级脚本、导出项目完整 SQL
    ├── sync.rs                   # 数据库同步：测试远程连接、获取远程表结构、比较本地与远程差异、生成同步 SQL、反向同步远程表/列/索引到本地
    └── git.rs                    # Git 集成：获取 Git 信息、初始化仓库、同步推送到远程仓库
```

## 关键模块说明

### 数据库方言 (`dialect.rs`)

通过 `DatabaseDialect` 和 `DatabaseConnector` 两个 trait 抽象数据库差异：

- **`DatabaseDialect`**：SQL 语法差异（AUTO_INCREMENT vs GENERATED ALWAYS AS IDENTITY、注释语法、DROP INDEX 语法、布尔字面量、类型映射等）
- **`DatabaseConnector`**：远程连接测试与表结构获取
- 工厂函数 `get_dialect()` / `get_connector()` 按 db_type 分派
- `get_supported_database_types` 命令供前端动态渲染数据库类型选项
- `get_type_mappings` 命令返回类型映射表（如 PG: tinyint→smallint, datetime→timestamp）

### 版本管理 (`version.rs`)

- 创建版本时快照当前所有表结构（含列、索引、初始数据）为 JSON
- 导出版本 SQL 和项目 SQL 时通过 dialect 适配不同数据库语法
- 版本间升级脚本自动 diff 表/列/索引/初始数据的增删改

### 数据库同步 (`sync.rs`)

- 通过 `DatabaseConnector` 连接远程 MySQL/PostgreSQL
- `compare_tables` 比较本地设计与远程实际表结构的差异
- `generate_sync_sql` 生成将本地设计同步到远程的 SQL
- 支持反向操作：将远程表结构同步回本地模型

### 本地存储

应用数据存储在 SQLite 数据库中（位于安装目录的 `data/` 文件夹），主要表：

| 表名 | 用途 |
|------|------|
| `t_project` | 项目 |
| `t_table` | 表定义 |
| `t_column` | 列定义 |
| `t_index` / `t_index_field` | 索引及索引字段 |
| `t_init_data` | 初始数据 |
| `t_version` | 版本快照 |
| `t_setting` | 键值设置（AI 配置、自定义类型等） |
| `t_database_connection` | 数据库连接配置 |

## 技术栈

| 层 | 技术 |
|----|------|
| 前端框架 | React 18 + TypeScript |
| UI 组件库 | Ant Design 5 |
| 路由 | React Router v6 |
| 构建工具 | Vite |
| 桌面框架 | Tauri v2 |
| 后端语言 | Rust |
| 本地数据库 | SQLite (rusqlite) |
| MySQL 驱动 | mysql crate |
| PostgreSQL 驱动 | postgres + native-tls |
| 自动更新 | tauri-plugin-updater |
