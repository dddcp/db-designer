## Context

DB Designer 当前仅有一份中文 `README.md`，位于仓库根目录。本次变更将其扩展为双语 README，以英文为默认展示语言，中文作为备选，同时提升整体视觉呈现质量。

## Goals / Non-Goals

**Goals:**
- 将 `README.md` 重写为英文，并保持与现有内容的信息对等
- 新建 `README.zh-CN.md` 承载中文内容
- 两个文件顶部均提供语言切换链接
- 美化 README 排版：优化徽章布局、标题层级、段落间距，使文档更专业易读

**Non-Goals:**
- 不更改截图（保留现有中文界面截图）
- 不修改除 README 以外的任何文档（如 CHANGELOG、AGENTS.md 等）
- 不涉及应用代码或功能逻辑变更

## Decisions

### 文件组织：根目录双文件方案
- **选择**：`README.md`（英文） + `README.zh-CN.md`（中文）
- **理由**：GitHub 原生支持多语言 README 展示，当用户访问仓库时自动按语言偏好匹配；同时根目录双文件是最直观的组织方式，无需额外配置

### 默认语言：英文优先
- **选择**：`README.md` 放英文，`README.zh-CN.md` 放中文
- **理由**：GitHub 默认渲染 `README.md`，国际用户访问时直接看到英文，中文用户可通过顶部链接切换

### 语言切换标识
- **选择**：使用国旗 Emoji + 文字链接，放置在文档顶部居中位置
- **示例**：`[🇺🇸 English](#) | [🇨🇳 简体中文](README.zh-CN.md)`
- **理由**：视觉直观，占用空间小，符合开源项目惯例

### 翻译策略：意译为主，保持技术准确性
- **选择**：不追求逐字直译，而是确保英文读者能准确理解产品能力
- **理由**：README 是营销+技术文档，可读性优先于字面一致性

### 美化方向
- 标题区采用居中布局，项目名称 + 一句话描述 + 核心卖点 + 徽章行
- 使用水平分割线（`---`）分隔大区块
- 功能表格统一左对齐，增强可读性
- 截图区域保持居中布局

## Risks / Trade-offs

- **[风险]** 未来更新 README 时需要同步维护两份文件 → **缓解**：README 变更频率较低；可在 AGENTS.md 中注明维护规范
- **[风险]** 英文翻译出现技术术语不准确 → **缓解**：关键术语（如 Tauri、React、AI 功能名）保留英文或提供英文对照
